Bricks-CLI

基于Ember-CLI项目之上，为Ember.js项目提供基本框架的命令行构建工具。让使用者快速地构建一个Ember.js项目模板，提高效率。

  • bricks help
  • bricks version
  • bricks update
  • bricks new
  • bricks init
  • bricks build
  • bricks generate
  • bricks test
  • bricks serve

Installing Bricks CLI

安装使用Bricks-CLI之前，你需要先安装Node，再用npm安装bricks-cli：

npm install -g bricks-cli

Bricks-CLI项目源码：

https://github.com/innobricks/bricks-cli.git

Using Bricks CLI

列出Bricks-CLI中的可用命令的帮助信息，通过 <options>可以指定查看对应命令的帮助信息，你可以省略或者用 h,-h,--help代替help：

bricks help <options>

输出Bricks-CLI的版本信息，你可以用v,-v,--version代替version：

bricks version

把Bricks-CLI更新到最新版本：

bricks update

为你在当前目录下创建一个名为<app-name>的文件夹并在文件夹中创建应用的结构：

bricks new <app-name>

在当前目录下创建应用结构，你可以用i代替init：

bricks init

构建应用，可以用bricks help build查看更多选项：

bricks build <options>

根据<blueprint>产生对应的项目文件，可以用bricks help generate查看更多关于<blueprint>的选项，你可以用g代替generate：

bricks generate <blueprint> <options>
如运行bricks generate lang xx会在应用项目中产生app/lang/xx.js文件用于项目国际化的应用。

运行应用目录下testem.json的测试（可添加任何需要测试的操作），你可以用t代替test：

bricks test

开启应用的服务器，默认的端口为4200，你可以用server,s代替serve：

bricks serve <options>

可以通过 <options>来设置对应的服务器参数,如下所示：

port：服务器端口，默认为4200，如运行bricks serve --port 8888可将端口改为8888
host：服务器IP地址，默认为0.0.0.0
output-path：输出路径，默认为dist/
live-reload：文件更新时是否自动重新加载，默认为true
...
